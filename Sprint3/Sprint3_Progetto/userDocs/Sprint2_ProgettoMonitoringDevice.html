Progetto

Software per led fisico acceso/spento/lampeggiante

Sono stati realizzati 3 file diversi che verranno chiamati dall'attore Led per eseguire le azioni fisiche corrispondenti.
Essendo il Led collegato ad un RaspBerry si è deciso di utilizzare il linguaggio Python con la libreria GPIO dato è la scelta più semplice e pratica per eseguire questo tipo di operazioni.
Files:
  1. Accensione -> LedOn.py
  2. Spegnimento -> LedOff.py
  3. Lampeggio -> LedBlink.py

Come controllare se l'ash storage è vuoto?

E' stata introdotta una variabile DMIN che identifica quando l'ash storage è vuoto, visto che non abbiamo un riferimento reale di quanto sia alto il container>.

Deployment

Per poter eseguire i due contesti nello stesso progetto è stato modificato il build.gradle in questo modo: 
	application {
		// Define the main class for the application.
		mainClassName = 'it.unibo.ctx_wis.MainCtx_wisKt'
		if (project.hasProperty("build_MD")) {
			mainClassName = 'it.unibo.ctx_monitoring_device.MainCtx_monitoring_deviceKt'
		}
	}
		
	jar {
		println("building jar")
		duplicatesStrategy(DuplicatesStrategy.EXCLUDE)  //per via di locgback.xml quando si usa spring
		from sourceSets.main.allSource
		from('./') {
			include '*.pl'
			include '*.json'
		}
		manifest {
			attributes 'Main-Class': "$mainClassName"
		}
	}
	distributions {
		main {
			distributionBaseName = 'wis'
			if (project.hasProperty("build_MD")) {
				distributionBaseName = 'monitoringdevice'
			}	  
			contents { 
					from './wis.pl'
					from './monitoringdevice.pl'
					from './sysRules.pl'
					from './facadeConfig.json'
					from './positions.pl'
					from './pointPicker.pl'
			}
		}
	}
Quando si vuole creare il Deployment per il WIS bisogna eseguire: 
gradlew distZip
Mentre per il Monitoring Device: 
gradlew distZip -Pbuild_MD
Per poter eseguire la demo del Core Business sono disponibili due file batch:
  • basicRobot.bat: avvia l'immagine docker del WEnv, imposta la stanza impostata nel file sceneConfig.js e apre una tab di chrome per la visualizzazione
  • wis.bat esegue il sistema WIS
  • monitoringdevice.bat esegue il sistema MonitoringDevice
NB: per eseguire correttamente il sistema, è necessaria la cartella unibolibs che contiene le librerie necessarie per l'esecuzione. Tale libreria va posizionata un livello sotto alla cartella dello Sprint1.
